# Pick A Side

تطبيق ويب ثابت يحاكي منصة Pick A Side لتحديات توقعات كرة القدم حسب المتطلبات:

- تسجيل الدخول وإنشاء الحساب ثم التحويل إلى `/`.
- لوحة رئيسية فيها الملف الشخصي، الكفاءة، المتابعون، بطولات المستخدم، وروابط Created / Joined / Draft / History.
- صفحة بحث مستقلة بأسلوب Instagram: البحث عن مستخدمين مع Follow، واستعراض بطولات Public قابلة للانضمام والبحث عنها بالاسم أو الكود.
- إنشاء بطولة مع الخصوصية، كود الدعوة، الدور الافتتاحي بما فيه دور المجموعات، الميزانية، والحد الأدنى للنقاط.
- منشئ البطولة يستطيع تفعيل ترشيحات اختيارية: أفضل لاعب، الهداف، أفضل حارس، وأفضل لاعب صاعد. اختيار اللاعب يتم بالبحث داخل قوائم اللاعبين الرسمية، ويمكن لاحقاً استبدال `officialRosterPlayers` بمصدر API حقيقي.
- صفحة بطولة فيها قفل الأدوار، توزيع النقاط، التحقق من الميزانية والحد الأدنى، وقفل التوقعات قبل 30 دقيقة من أول مباراة في الدور.
- اللاعب الذي لا يرسل توقعاته قبل القفل يخسر ميزانية الدور، وتدخل نقاطه في وعاء الجولة ليتم توزيعها على الفائزين عند التسوية.
- صفحة البطولة تحتوي على `Pick Board` عام: اختيار طرف كل مباراة ثم توزيع النقاط ومراجعة الترشيحات، بدون منطق خاص بكأس العالم أو المجموعات.
- شاشة مباشر تعرض فقط البطولات النشطة التي يشارك فيها المستخدم، وعند اختيار بطولة تظهر نتائجها الحية وتأثيرها على رصيد المستخدم.
- دالة `calculateJackpot` داخل `app.js` تمثل منطق الـ backend cron-job لتوزيع وعاء نقاط الخاسرين على الفائزين بالنسبة والتناسب.
- ملف `jackpot-engine.js` يحتوي نسخة مستقلة من محرك التسوية لاستخدامها لاحقاً داخل Backend Cron Job.

## التشغيل

افتح الملف `index.html` في المتصفح. لا توجد حزم أو خطوات تثبيت لأن المشروع مكتوب بـ HTML/CSS/JS فقط.

## ملاحظات الربط الحقيقي

البيانات الحالية محاكاة داخل `app.js`. عند تحويله إلى منتج فعلي، استبدل كائن `state` بطبقة API تشمل:

- Auth API للمستخدم والجلسة.
- Users API للبحث والمتابعة.
- Tournaments API للإنشاء والمسودات والتفاصيل.
- Sports API للمباريات والنتائج.
- Cron/Worker يستدعي `calculateJackpot` بعد نهاية كل مباراة ويحدّث الـ leaderboard.
